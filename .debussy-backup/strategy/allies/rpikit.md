# rpikit

**URL:** https://github.com/bostonaholic/rpikit
**Category:** Research-Plan-Implement development framework

## What It Does

Claude Code plugin implementing the Research-Plan-Implement (RPI) framework
with human approval gates between phases.

## Relationship

Methodological ally — rpikit enforces research-first discipline similar to
Debussy's strategy/roadmap skills. rpikit focuses on per-feature engineering
(research codebase -> plan implementation -> implement with TDD); Debussy
focuses on product-level strategy and multi-step workflow orchestration.

## Integration Opportunity

- rpikit's RPI phases could be individual steps within a Debussy workflow-run
  YAML, gaining browser-based review UX and monitoring
- Debussy's strategy artifacts (problems, audiences) could inform rpikit's
  research phase
- rpikit's code review and security review skills complement Debussy's
  feedback skill for different review surfaces

## Synergies

- Both emphasize human oversight at critical transitions
- Complementary scopes: feature-level (rpikit) vs. product-level (Debussy)
- Both are installed in the same Debussy plugin — already coexisting
- rpikit's brainstorming skill fills a gap Debussy doesn't address
