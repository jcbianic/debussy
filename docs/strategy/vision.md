# Vision: Debussy

## Why We're Building This

Claude Code is awesome. The more you use it, the faster you move — and the
faster you move, the more threads you pull at once. You research a feature
while planning another while reviewing a third. The velocity is real, but it
creates **cognitive debt**: the growing gap between what's happening in your
project and what you can keep track of.

Debussy is a solo experiment to manage that cognitive debt. It provides
structured workflows and meta-workflows with a simple UI that surfaces what's
going on in your project and what requires your attention. Review and
observability are first-class citizens — not afterthoughts squeezed into a
terminal.

Debussy is in an **experimental phase**. The skills are being dogfooded on
Debussy itself — strategy, roadmap, feedback, and workflow-run are all used
to build and refine Debussy.

## North Star

Keep velocity high while managing the cognitive debt of pushing Claude Code
to its limits. Remain agnostic to methodology and framework — Debussy is
plumbing, not opinion.

## Success Criteria

- Skills and browser UIs become a natural part of the daily Claude Code
  routine — used without thinking about setup or configuration
- The strategy→roadmap pipeline produces artifacts that flow into implementation
  without manual reformatting
- Review workflows feel faster and more thorough than reading terminal output
- The plugin works end-to-end on Debussy itself (dogfooding as validation)
