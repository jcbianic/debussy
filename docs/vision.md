# Vision: Debussy

## Why We're Building This
Claude Code is powerful but its default interaction model — reading checklists,
watching terminals scroll, manually switching contexts — creates invisible
friction. Debussy exists to eliminate that friction for developers who live in
Claude Code daily. Every skill targets a specific moment where the default
experience breaks down.

## Pain Points

### 1. Review UX
Reading through 20-item checklists while typing responses is exhausting and
error-prone. There's no spatial separation between reading and responding. A
structured browser UI — items one at a time, keyboard-navigable, returning
machine-readable answers — removes this bottleneck.

### 2. Long Workflow Monitoring
Kicking off a multi-step workflow and not knowing where it is, how far it's
gone, or what it's waiting on creates anxiety and interruptions. Unattended
execution requires visible, trustworthy progress state.

### 3. Parallel Lanes
Working on multiple independent subjects simultaneously in Claude Code creates
conflicts: shared context, shared git working tree, shared tool state. There's
no natural way to isolate parallel work. Git worktrees offer the primitive;
Debussy can make them first-class.

## What Success Looks Like
In 6 months: all three pain points are solved for the primary developer's own
workflow. The plugin is used daily without thinking about it. Each skill is
stable enough that others can install it and get value without hand-holding.

## North Star
Zero friction: the plugin disappears into the workflow. No setup overhead, no
unexpected failures, no manual workarounds.
