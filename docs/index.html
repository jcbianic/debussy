Orchestrate Claude Code.
Without the chaos.

Debussy is a local-first web UI for managing Claude Code sessions, running structured multi-phase workflows, and reviewing AI-produced artifacts — all from the browser, without juggling terminal tabs.

npx debussy

The problem

Three concrete friction points in daily Claude Code use — none of the existing tools address all three together.

01

Review UX is broken

Reading a 20-item checklist in a terminal while typing responses is high friction. No way to approve, flag, or skip with a click.

02

Long workflows need visibility

A full IIKit cycle takes 30–60 minutes. You shouldn't babysit it — but you can't leave it blind either. You need a clear picture when you return.

03

Parallel work creates chaos

Running two features simultaneously means filesystem conflicts and context-switching mess. There's no built-in concept of isolated lanes.

What Debussy adds

⬡

Session Dashboard

Browse, search, and resume Claude Code sessions from the browser. Real-time status across multiple concurrent sessions. Permission prompts surfaced in the UI.

▶

Workflow Runner

Execute IIKit phases, Ralph loops, and custom pipelines from the UI. Visualize phase progression, auto-switch on exit criteria, and surface validation results at each checkpoint.

◈

Artifact Manager

View, edit, and validate artifacts produced by Claude Code — specs, plans, task lists, test definitions. Review what each phase produced and validate assertion integrity.

◎

Cost & Monitoring

Track token consumption, burn rate, and cost per session and workflow. Surface context window usage in real-time. Alert when sessions approach limits.

Quick start

Run Debussy from any repository. No install required — it opens at localhost:3333 and reads your existing Claude Code session history.

# Run from any repository npx debussy   # Opens http://localhost:3333

Local-first — no cloud, no telemetry, no accounts. Session data is read from ~/.claude/projects/. Debussy's own metadata lives in ~/.debussy/.
