The Three Pain Points

1. Review UX

Reading through checklists while typing responses is friction-heavy. You need a clean UI to review items one by one and produce structured answers with minimal effort.

2. Workflow Monitoring

Long workflows running unattended leave you in the dark. You need clear visibility into progress and your position within a multi-step process at any time.

3. Parallel Work

Juggling multiple subjects in one session creates conflicts. You need isolated lanes to work on different things simultaneously without making a mess.

Debussy tackles these with a plugin ecosystem. Two skills to start. Many more coming.

Getting Started

Feedback Skill

Collect structured user feedback via an interactive browser UI. Present a list of items, let users approve, reject, or discuss each one, and get back structured responses.

  • Browser-based review interface
  • Configurable actions and workflows
  • No token consumption while reviewing
  • Structured JSON responses
/feedback request.json

Workflow-Run Skill

Execute multi-step workflows with interactive review gates. Run steps, collect artifacts, present review UI, and block on user decisions.

  • Multi-step YAML workflows
  • Artifact collection and review
  • Interactive decision gates
  • Real-time progress dashboard
/workflow-run workflow.yml

Installation

Clone and link the plugin locally to your Claude Code setup

git clone https://github.com/jcbianic/debussy
cd debussy
npm link

The plugin installs two skills: feedback and workflow-run
Use them with /feedback and /workflow-run commands
